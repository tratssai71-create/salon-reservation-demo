# サロン予約システム（プロトタイプ）

メニュー→日付→時間→スタッフ→お客様情報→確認→完了、というホットペッパー的な予約フローを持つ独自の予約システムです。予約が確定すると、Googleカレンダーに自動でイベントが作成され、お客様に確認メールが送信されます。管理画面（`admin.html`）から、予約の確認・時間のブロック・メニュー/スタッフ/営業時間の変更ができます。

## 構成

| ファイル | 役割 |
|---|---|
| `index.html` / `style.css` / `app.js` | フロントエンド（お客様向け予約フォーム） |
| `admin.html` / `admin.css` / `admin.js` | 管理画面（予約確認・時間ブロック・メニュー/スタッフ/営業時間・パスワード変更） |
| `config.js` | デモモード用の初期表示内容と、API接続先（`API_URL`）の設定 |
| `gas/Code.gs` | バックエンド（Google Apps Script）。カレンダー登録・メール送信・管理画面API |
| `gas/appsscript.json` | GASのマニフェスト（デプロイ設定） |

## 今すぐ試す（デモモード）

`config.js` の `API_URL` が空文字のままだと、実際のカレンダーに接続せず、ブラウザ内だけで予約フローを最後まで試せる「デモモード」で動作します。`index.html` を開くだけで確認できます（この状態では `admin.html` は利用できません）。

## 本番接続（Googleカレンダーに自動追加する）

### 1. 予約用のGoogleカレンダーを用意する

- サロンで使うGoogleアカウントにログインし、Googleカレンダーで新しいカレンダーを作成（または既存のメインカレンダーを利用）
- 別カレンダーを使う場合は、そのカレンダーの「設定と共有」→「カレンダーの統合」から **カレンダーID** をコピーし、`gas/Code.gs` 冒頭の `CALENDAR_ID` に設定する（メインカレンダーでよければ `"primary"` のままでOK）

### 2. Google Apps Script プロジェクトを作成する

1. [script.google.com](https://script.google.com/) にアクセスし、「新しいプロジェクト」を作成
2. デフォルトの `Code.gs` の中身を、このフォルダの `gas/Code.gs` の内容で置き換える
3. 左メニューの歯車アイコン（プロジェクトの設定）→「`appsscript.json` マニフェスト ファイルをエディタで表示する」にチェックを入れる
4. `appsscript.json` の中身を、このフォルダの `gas/appsscript.json` の内容で置き換える
5. 必要であれば `Code.gs` 冒頭の定数を書き換える
   - `CALENDAR_ID`: 手順1で用意したカレンダーID
   - `NOTIFY_EMAIL`: 予約が入るたびに通知を受け取りたいメールアドレス（不要なら空文字のまま）
   - `SPREADSHEET_ID`: 予約ログを記録したいスプレッドシートのID（不要なら空文字のまま）
   - `DEFAULT_ADMIN_PASSWORD`: 管理画面の初期パスワード（**デプロイ後、必ず管理画面から変更してください**）

店舗名・メニュー・スタッフ・営業時間・定休日は、コードに直接書く必要はありません。初回アクセス時に `DEFAULT_CONFIG` の内容が自動保存され、以降は管理画面から変更できます。

### 3. ウェブアプリとしてデプロイする

1. 画面右上の「デプロイ」→「新しいデプロイ」
2. 種類の選択で「ウェブアプリ」を選ぶ
3. 「次のユーザーとして実行」: 自分（＝カレンダーの持ち主のアカウント）
4. 「アクセスできるユーザー」: 全員
5. 「デプロイ」をクリックし、初回は権限の承認（カレンダー・Gmail・スプレッドシートへのアクセス許可）を行う
6. 発行された **ウェブアプリのURL**（`https://script.google.com/macros/s/xxxx/exec`）をコピー

### 4. フロントエンドに接続する

`config.js` を開き、`API_URL` に手順3で取得したURLを設定する:

```js
API_URL: "https://script.google.com/macros/s/xxxxxxxxxx/exec",
```

保存して `index.html` を開き直すとデモバナーが消え、実際にGoogleカレンダーへ予約が登録されるようになります。

### 5. 管理画面にログインする

`admin.html` を開き、初期パスワード（`gas/Code.gs` の `DEFAULT_ADMIN_PASSWORD`）でログインします。ログイン後、「店舗設定」タブから**必ずパスワードを変更**してください。

管理画面でできること:
- **予約一覧**: 直近60日分の予約・ブロックの確認と削除
- **時間をブロック**: 休憩・私用・研修などで受付を止めたい時間帯を指定して登録（サイト上では自動的に×表示になります）
- **メニュー**: カテゴリ・メニュー項目（名前／所要時間／料金／説明）の追加・編集・削除
- **スタッフ**: スタッフの追加・編集・削除
- **営業時間**: 開店・閉店時刻、予約枠の刻み幅、定休日の変更
- **店舗設定**: 店舗名・キャッチコピー・電話番号・住所、管理者パスワードの変更

これらの変更はGAS側の保存領域（PropertiesService）に即座に反映され、コードの再デプロイは不要です。

### 6. コード修正後の再デプロイ

`Code.gs` 自体（ロジック）を修正した場合のみ、「デプロイ」→「デプロイを管理」→ 該当のデプロイの鉛筆アイコン →「バージョン」で「新バージョン」を選んで再デプロイが必要です。メニューやスタッフなどの内容変更は管理画面から行えるため、この操作は不要です。

## 個別の時間帯だけ空けたい／塞ぎたい場合

予約可否は毎回リアルタイムでGoogleカレンダーの実際の予定を見て計算します。管理画面の「時間をブロック」を使わなくても、Googleカレンダーアプリで直接その時間に予定を入れれば、同様にその時間は自動的に予約不可（×）になります。

## カスタマイズ

- 配色は `style.css` / `admin.css` 冒頭の `:root` 内のCSS変数（`--ac` など）で調整できます
- デモモード時の初期表示内容（`config.js`）と、本番の初期コンテンツ（`gas/Code.gs` の `DEFAULT_CONFIG`）は別管理です。本番デプロイ後は管理画面での変更が優先されます
